========================================================================
                      AGRIVISION AI - RUN & DEPLOYMENT NOTES
========================================================================

This file outlines the steps required to configure, run locally, and 
deploy the AgriVision AI backend and mobile application.

------------------------------------------------------------------------
1. LOCAL SYSTEM REQUIREMENTS
------------------------------------------------------------------------
- Python 3.10+
- Node.js 18+ & npm
- Git
- Expo Go app installed on a physical mobile device (for native testing)

========================================================================
PART 1: LOCAL RUNNING & SETUP
========================================================================

------------------------------------------------------------------------
A. BACKEND SETUP (FastAPI)
------------------------------------------------------------------------
1. Navigate to the backend directory:
   cd backend

2. Create a Python Virtual Environment:
   python -m venv venv

3. Activate the Virtual Environment:
   - On Windows PowerShell:
     .\venv\Scripts\Activate.ps1
   - On Windows CMD:
     .\venv\Scripts\activate.bat
   - On macOS/Linux:
     source venv/bin/activate

4. Install dependencies:
   pip install -r requirements.txt

5. Configure Environment Variables:
   Create a `.env` file in the `backend` directory (copy from `.env.example` if available).
   It should contain:
   ------------------------------------------------------------
   PORT=8000
   HOST=0.0.0.0
   SUPABASE_URL=https://<your-supabase-project-id>.supabase.co
   SUPABASE_KEY=<your-supabase-anon-key>
   FIREBASE_API_KEY=<your-firebase-api-key>
   FIREBASE_AUTH_DOMAIN=<your-firebase-auth-domain>
   FIREBASE_PROJECT_ID=<your-firebase-project-id>
   GEMINI_API_KEY=<your-google-gemini-api-key>
   ------------------------------------------------------------

6. Run the local backend server:
   python -m uvicorn main:app --host 0.0.0.0 --port 8000
   
   - Swagger API Docs will be available at: http://127.0.0.1:8000/docs
   - The `--host 0.0.0.0` flag is critical to allow connection from physical 
     mobile devices on the same Wi-Fi network.

------------------------------------------------------------------------
B. MOBILE FRONTEND SETUP (React Native & Expo)
------------------------------------------------------------------------
1. Navigate to the mobile directory:
   cd mobile

2. Install dependencies:
   npm install

3. Configure Environment Variables:
   Create a `.env` file in the `mobile` directory. It must contain:
   ------------------------------------------------------------
   EXPO_PUBLIC_API_URL=http://<YOUR_LOCAL_IP_ADDRESS>:8000
   EXPO_PUBLIC_SUPABASE_URL=https://<your-supabase-project-id>.supabase.co
   EXPO_PUBLIC_SUPABASE_KEY=<your-supabase-anon-key>
   EXPO_PUBLIC_FIREBASE_API_KEY=<your-firebase-api-key>
   EXPO_PUBLIC_FIREBASE_AUTH_DOMAIN=<your-firebase-auth-domain>
   EXPO_PUBLIC_FIREBASE_PROJECT_ID=<your-firebase-project-id>
   EXPO_PUBLIC_FIREBASE_STORAGE_BUCKET=<your-firebase-storage-bucket>
   EXPO_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=<your-sender-id>
   EXPO_PUBLIC_FIREBASE_APP_ID=<your-app-id>
   ------------------------------------------------------------
   
   IMPORTANT:
   - For Android Emulators connecting to a backend running on the SAME computer, 
     use: EXPO_PUBLIC_API_URL=http://10.0.2.2:8000
   - For iOS Simulators connecting to the same computer, 
     use: EXPO_PUBLIC_API_URL=http://127.0.0.1:8000
   - For Physical Devices running via Expo Go:
     Get your computer's local network IP (run `ipconfig` on Windows or `ifconfig` 
     on macOS/Linux). Use that IP address (e.g., http://192.168.55.146:8000). Both 
     your phone and computer must be connected to the exact same Wi-Fi router.

4. Start the Expo Dev Server with cache cleared:
   npx expo start -c --go

5. Open the App:
   - Scan the QR code printed in the terminal using the Expo Go app (Android) 
     or the iOS Camera App (iOS).


========================================================================
PART 2: DATABASE SETUP (Supabase)
========================================================================
Execute the following SQL queries inside the Supabase SQL Editor to set up 
the tables and storage buckets required for AgriVision AI:

------------------------------------------------------------------------
1. Create Predictions Table:
------------------------------------------------------------------------
CREATE TABLE predictions (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id VARCHAR(255) NOT NULL,
  disease VARCHAR(255) NOT NULL,
  confidence FLOAT NOT NULL,
  image_url TEXT,
  language VARCHAR(50) DEFAULT 'English',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

------------------------------------------------------------------------
2. Enable Storage Buckets:
------------------------------------------------------------------------
- Go to "Storage" in the Supabase Dashboard.
- Create a new public bucket named `predictions`.
- Set policies to allow all users (or authenticated users) to upload 
  and read files.


========================================================================
PART 3: DEPLOYMENT GUIDES
========================================================================

------------------------------------------------------------------------
A. BACKEND DEPLOYMENT TO RENDER
------------------------------------------------------------------------
Render is an excellent cloud hosting service that supports FastAPI and 
custom Python environments.

1. Prepare your GitHub Repository:
   Commit your backend code. Make sure that:
   - `requirements.txt` is up to date.
   - The large `.keras` model file (`mobilenet_crop_disease.keras`) is committed.
     * Note: If Git rejects the file because it is > 20MB, configure Git LFS:
       git lfs install
       git lfs track "*.keras"
       git add .gitattributes mobilenet_crop_disease.keras
       git commit -m "Add model via LFS"

2. Create a Render Web Service:
   - Log in to https://dashboard.render.com/ and click "New" -> "Web Service".
   - Connect your GitHub repository containing the backend code.

3. Configure Web Service Settings:
   - Name: `agrivision-backend`
   - Region: Select a region closest to your target users (e.g., Singapore/US).
   - Runtime: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python -m uvicorn main:app --host 0.0.0.0 --port $PORT`
   - Instance Type: Choose `Free` (or a paid instance if memory usage during TensorFlow 
     model execution exceeds 512 MB).

4. Environment Variables on Render:
   In the Render dashboard under the "Environment" tab, add all environment variables 
   found in the backend `.env` file (excluding PORT and HOST, which Render manages dynamically):
   - `SUPABASE_URL`
   - `SUPABASE_KEY`
   - `FIREBASE_API_KEY`
   - `FIREBASE_AUTH_DOMAIN`
   - `FIREBASE_PROJECT_ID`
   - `GEMINI_API_KEY`

5. Deploy:
   Render will automatically pull the code, install dependencies, load the 
   MobileNetV2 model, and expose a public URL (e.g., https://agrivision-backend.onrender.com).

------------------------------------------------------------------------
B. ALTERNATIVE BACKEND DEPLOYMENT: DOCKER / AWS / GCP
------------------------------------------------------------------------
If Render free instance runs out of RAM during TensorFlow startup, deploy 
via Docker to Google Cloud Run or AWS EC2:

1. Create a `Dockerfile` in `backend/`:
   ------------------------------------------------------------
   FROM python:3.10-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   EXPOSE 8000
   CMD ["python", "-m", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
   ------------------------------------------------------------

2. Build and run locally:
   docker build -t agrivision-backend .
   docker run -p 8000:8000 --env-file .env agrivision-backend

3. Push to Google Container Registry (GCR) or AWS ECR, and instantiate 
   on Google Cloud Run or AWS ECS.

------------------------------------------------------------------------
C. FRONTEND DEPLOYMENT TO GOOGLE PLAY STORE / IOS APP STORE (EAS BUILD)
------------------------------------------------------------------------
Expo Application Services (EAS) builds native binaries (APKs/IPAs) in the cloud.

1. Install EAS CLI globally:
   npm install -g eas-cli

2. Log in to your Expo account:
   eas login

3. Initialize EAS Configuration:
   Go to the `mobile` directory and run:
   eas build:configure

4. Verify `eas.json` file. Ensure there is a profile to generate an APK (for testing):
   ------------------------------------------------------------
   {
     "cli": {
       "version": ">= 9.0.0"
     },
     "build": {
       "development": {
         "developmentClient": true,
         "distribution": "internal"
       },
       "preview": {
         "distribution": "internal",
         "android": {
           "buildType": "apk"
         }
       },
       "production": {}
     },
     "submit": {
       "production": {}
     }
   }
   ------------------------------------------------------------

5. Update Mobile Environment Variables for Production:
   Replace `EXPO_PUBLIC_API_URL` in `mobile/.env` with your newly deployed public backend 
   URL (e.g., https://agrivision-backend.onrender.com).

6. Run the Native Build:
   - For Android (generates a installable `.apk` file for demo devices):
     eas build --platform android --profile preview
   
   - For App Store submission (generates a `.aab` file):
     eas build --platform android --profile production

7. Download and Distribute:
   Once the cloud build completes, EAS will print a download link to your `.apk` or `.aab`. 
   Share the `.apk` directly with stakeholders or submit the `.aab` to the Google Play Console.
